# 🌾 מכרזים חקלאיים · אנגל — אזור עמק יזרעאל

מערכת שרצה בענן (חינם), עוברת על מכרזים פתוחים רלוונטיים לחווה, ושולחת:
- **מייל יומי** על מכרזים חדשים — עם שתי כותרות: "מכרזים חדשים" ו"מכרזים רלוונטיים השבוע".
- **מייל סיכום שבועי** (יום ראשון) עם כל המכרזים של השבוע + **קובץ אקסל מעוצב** מצורף.
- **קובץ אקסל** (`tenders.xlsx`) שמתעדכן אוטומטית, ובו כל המכרזים מסודרים לפי חודשים.

> **כרגע המערכת מסננת רק חיטה / גד"ש וחקלאות כללית — בלי מרעה בקר.**
> להחזרת הבקר: הוסף `"grazing"` ל-`EMAIL_BUCKETS` ב-`config.py`.

## מה היא סורקת

| מקור | מה מגיע משם |
|------|-------------|
| מ.א. עמק יזרעאל, הגלבוע, מגידו | מכרזי המועצות (כבר באזור בהגדרה) |
| מינהל הרכש הממשלתי (mr.gov.il) | **רמ"י** (השכרה עונתית / עיבוד), משרד החקלאות, רשות הטבע והגנים, בתי ספר חקלאיים ממשלתיים |

הסינון: מילות מפתח חקלאיות (בקבוצות נפרדות) + סינון גאוגרפי לפי שם היישוב, בהתאמת מילה שלמה.

## הרצה מקומית לבדיקה

```bash
pip3 install -r requirements.txt
python3 main.py --dry-run      # מדפיס מה נמצא, בלי מייל ובלי שמירה
python3 main.py --preview      # מייצר email_preview.html + tenders.xlsx לצפייה
```

## התקנה מלאה (רץ לבד)

### 1. סיסמת אפליקציה ל-Gmail
1. הפעל **אימות דו-שלבי**: https://myaccount.google.com/security
2. צור **App Password**: https://myaccount.google.com/apppasswords → "Mail" → העתק 16 תווים.

### 2. העלאה ל-GitHub
1. צור **repository פרטי** ב-https://github.com/new (למשל `engel-farm-tenders`).
2. מהתיקייה הזו:
   ```bash
   git remote add origin https://github.com/<USERNAME>/engel-farm-tenders.git
   git branch -M main
   git push -u origin main
   ```

### 3. הגדרת הסודות (Secrets)
**Settings → Secrets and variables → Actions → New repository secret**:

| שם הסוד | ערך |
|---------|-----|
| `SMTP_USER` | כתובת ה-Gmail השולחת, למשל `pninasweb@gmail.com` |
| `SMTP_APP_PASSWORD` | 16 התווים מסעיף 1 (בלי רווחים) |
| `MAIL_TO` | לאן לשלוח (אפשר כמה, מופרד בפסיק). אם ריק — נשלח לעצמך |

### 4. הפעלה
1. לשונית **Actions** → אשר הפעלת Workflows.
2. "מכרזים חקלאיים יומי" → **Run workflow** להרצה ראשונה.
3. מכאן: היומי רץ כל בוקר, והשבועי כל יום ראשון — אוטומטית.

## התאמות נפוצות (`config.py`)

- **להחזיר/להסתיר בקר** → `EMAIL_BUCKETS` (הוסף/הסר `"grazing"`)
- **להוסיף/להסיר יישוב מהרדיוס** → `REGION_SETTLEMENTS`
- **מילות מפתח** → `WHEAT_KEYWORDS` / `GRAZING_KEYWORDS` / `GENERAL_AGRI_KEYWORDS`
- **להוסיף מועצה אזורית** → `COUNCILS`
- **שעות השליחה** → שדות ה-`cron` ב-`.github/workflows/daily.yml` ו-`weekly.yml`

## מבנה הקוד

```
main.py             תזמור: איסוף → סינון → מסד → מייל (יומי/שבועי)
config.py           כל ההגדרות (מקורות, מילים, יישובים, קבוצות)
sources/councils.py גרידת המועצות האזוריות
sources/mr_gov.py   גרידת מינהל הרכש (רמ"י ומשרד החקלאות)
classify.py         סיווג לקבוצה + סינון אזור (התאמת מילה שלמה)
state.py            מסד המכרזים (data.json) עם תאריך גילוי
notify.py           בניית מיילים HTML ושליחה דרך Gmail
spreadsheet.py      יצירת קובץ האקסל המעוצב (לפי חודשים)
```

## מגבלות ידועות

- **קק"ל עדיין לא מחובר** — האתר שלהם בנוי על JavaScript/postback (ASP.NET) וללא API,
  כך שגרידה אמינה דורשת דפדפן headless. יתווסף כשנחזיר את המרעה.
- הסינון הגאוגרפי מבוסס על רשימת יישובים ב-`config.py` — יישוב שלא ברשימה לא ייתפס.
- אתר רמ"י מפרסם לפעמים עסקה עם כמה יישובים; המערכת מסננת לפי היישוב הראשי.
- אם מועצה תשנה את מבנה האתר, ייתכן שיידרש עדכון קטן ב-`sources/councils.py`.
