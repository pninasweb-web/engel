# 🌾 התראות מכרזים חקלאיים — אזור עמק יזרעאל

מערכת שרצה כל בוקר בענן (חינם), עוברת על מכרזים פתוחים רלוונטיים לחווה,
ושולחת מייל **רק על מכרזים חדשים** — מחולקים לשתי קבוצות: חיטה/גד"ש ומרעה בקר.

## מה היא סורקת

| מקור | מה מגיע משם |
|------|-------------|
| מ.א. עמק יזרעאל, הגלבוע, מגידו | מכרזי המועצות (כבר באזור בהגדרה) |
| מינהל הרכש הממשלתי (mr.gov.il) | **רמ"י** (השכרה עונתית / מרעה / עיבוד), משרד החקלאות, רשות הטבע והגנים, בתי ספר חקלאיים ממשלתיים |

הסינון: מילות מפתח חקלאיות בשתי קבוצות נפרדות + סינון גאוגרפי לפי שם היישוב
(רלוונטי למקורות הארציים; ניתן לעריכה ב-`config.py`).

## הרצה מקומית לבדיקה

```bash
pip3 install -r requirements.txt
python3 main.py --dry-run      # מדפיס מה נמצא, בלי לשלוח מייל ובלי לשמור מצב
```

## התקנה מלאה (רץ לבד כל יום)

### 1. סיסמת אפליקציה ל-Gmail
כדי שהמערכת תוכל לשלוח מייל מהחשבון שלך:
1. הפעל **אימות דו-שלבי** בחשבון Google (חובה): https://myaccount.google.com/security
2. צור **App Password**: https://myaccount.google.com/apppasswords → בחר "Mail" → העתק את 16 התווים.

### 2. העלאה ל-GitHub
1. צור **repository פרטי** חדש ב-https://github.com/new (למשל `agri-tenders`).
2. מהתיקייה הזו הרץ:
   ```bash
   git remote add origin https://github.com/<USERNAME>/agri-tenders.git
   git branch -M main
   git push -u origin main
   ```

### 3. הגדרת הסודות (Secrets)
ב-GitHub: **Settings → Secrets and variables → Actions → New repository secret**.
צור שלושה:

| שם הסוד | ערך |
|---------|-----|
| `SMTP_USER` | כתובת ה-Gmail השולחת, למשל `pninasweb@gmail.com` |
| `SMTP_APP_PASSWORD` | 16 התווים מסעיף 1 (בלי רווחים) |
| `MAIL_TO` | לאן לשלוח (אפשר כמה, מופרד בפסיק). אם ריק — נשלח לעצמך |

### 4. הפעלה
1. לשונית **Actions** → אשר הפעלת Workflows.
2. בחר את "מכרזים חקלאיים יומי" → **Run workflow** להרצה ראשונה ידנית ובדיקה.
3. מכאן זה רץ אוטומטית כל בוקר.

## התאמות נפוצות (`config.py`)

- **להוסיף/להסיר יישוב מהרדיוס** → `REGION_SETTLEMENTS`
- **להוסיף מילת מפתח** → `WHEAT_KEYWORDS` / `GRAZING_KEYWORDS` / `GENERAL_AGRI_KEYWORDS`
- **להוסיף מועצה אזורית** → `COUNCILS` (כל עוד היא על אותה פלטפורמה — `/bids/`)
- **שעת השליחה** → שדה ה-`cron` ב-`.github/workflows/daily.yml`

## מבנה הקוד

```
main.py            תזמור: איסוף → סינון → זיכרון → מייל
config.py          כל ההגדרות (מקורות, מילים, יישובים)
sources/councils.py גרידת המועצות האזוריות
sources/mr_gov.py   גרידת מינהל הרכש (רמ"י ומשרד החקלאות)
classify.py        סיווג לקבוצה + סינון אזור (התאמת מילה שלמה)
state.py           זיכרון מכרזים שכבר טופלו (seen.json)
notify.py          בניית מייל HTML ושליחה דרך Gmail
```

## מגבלות ידועות

- הסינון הגאוגרפי מבוסס על רשימת יישובים ב-`config.py` — אם מכרז נמצא ביישוב
  שלא ברשימה, הוא לא ייתפס. קל להרחיב את הרשימה.
- אתר רמ"י מפרסם לפעמים עסקה שכוללת כמה יישובים; המערכת מסננת לפי היישוב הראשי.
- אם מועצה תשנה את מבנה האתר שלה, ייתכן שיידרש עדכון קטן ב-`sources/councils.py`.
